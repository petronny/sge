NAME

       busy-nodes, idle-nodes, dead-nodes, nodes-in-job, qselect-node-list -
       List Grid Engine node names by various criteria

SYNOPSIS

       busy-nodes [-w] [-h|--help]
       idle-nodes [-w] [-h|--help]
       dead-nodes [-w] [-h|--help]
       nodes-in-job [-w] [-h|--help] jobid
       qselect-node-list [-h|--help] qselect_arg ...

DESCRIPTION

       busy-nodes, idle-nodes, and dead-nodes respectively list Grid Engine
       nodes which are:  busy with one or more jobs, idle (without a job
       running), and "dead" (their Grid Engine exec daemon is not responding).
       nodes-in-job list nodes which are running the given jobid (presumably a
       parallel one).  qselect-node-list runs qselect(1) with the given
       arguments.  The node list is printed one node per line unless the -w
       option is used.

OPTIONS

       -h,--help
              Print usage information.

       -w     Print the output in a form compatible with the -w argument of
              pdsh(1).

EXAMPLES

       The following would reboot all idle nodes.  (It is actually preferable
       to submit a job to reboot nodes, to avoid a race with job further
       submission.)
              pdsh -w `idle-nodes -w` reboot

       The following would run qhost(1), selecting only nodes in job number
       "123".
              qhost `nodes-in-job 123`

SEE ALSO

       qhost(1), qselect(1), pdsh(1).
